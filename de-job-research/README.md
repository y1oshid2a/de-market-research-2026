# データエンジニア求人調査リポジトリ
## メガベンチャー・外資IT 対象

---

## 目的

メガベンチャーおよび外資IT企業のデータエンジニア（DE）求人を体系的に収集・分析し、
以下の問いに答えることを目指す。

- 各社が求めるスキル・技術スタックの共通項と差異は何か
- 年収・待遇の相場はどの程度か
- キャリアパス（ポジション名・グレード）はどう設計されているか
- 日本語／英語要件、リモート可否などの就業条件の傾向は何か

## 調査方針

### 1. 情報収集

- **一次ソース優先**: 各社公式採用ページ（Careers / Jobs）から直接取得
- **補助ソース**: LinkedIn Jobs、Glassdoor、OpenWork、Wantedly、Indeed Japan
- **スナップショット管理**: 取得日を必ず記録する（求人は随時変更されるため）
- **重複排除**: 同一ポジションが複数媒体に掲載されている場合、公式ページを正とする

### 2. 記録フォーマット

各求人は `templates/job_template.md` に従い Markdown で記録する。
構造化データとして `jobs/processed/` に CSV も合わせて管理する。

### 3. 分析観点

収集した求人から以下の軸で比較分析を行う。

| 軸 | 具体的な項目 |
|----|-------------|
| 技術スタック | 言語(Python/SQL/Scala)、DWH、オーケストレーター、可視化ツール |
| クラウド | AWS / GCP / Azure / マルチクラウド |
| データ基盤 | Lakehouse、Streaming、Batch ETL |
| 求められる経験年数 | 0-2 / 3-5 / 5+ 年 |
| 年収レンジ | 提示額（税前・年収）または市場推定 |
| 勤務形態 | フルリモート / ハイブリッド / 出社 |
| 英語要件 | 不要 / ビジネスレベル / ネイティブレベル |

### 4. 更新頻度

- 新規求人の追加：随時
- 分析サマリーの更新：月1回を目安

## ディレクトリ構成

```
de-job-research/
├── README.md                  # 本ファイル（調査方針）
├── companies/
│   ├── mega-ventures/         # 各メガベンチャーの企業プロファイル
│   └── foreign-it/            # 各外資IT企業のプロファイル
├── jobs/
│   ├── raw/                   # 取得したままの求人テキスト（加工前）
│   └── processed/             # 正規化・整形済みの求人データ (CSV / MD)
├── analysis/                  # 比較分析・サマリーレポート
└── templates/
    └── job_template.md        # 求人記録テンプレート
```

---

## 使い方

### 新しい求人を追加する

1. `templates/job_template.md` をコピーして `jobs/raw/<企業名>_<YYYYMMDD>.md` として保存
2. テンプレートの各フィールドを埋める
3. 整形後 `jobs/processed/` にも同内容をCSV行として追記する

### 分析を更新する

`analysis/` 配下に `summary_YYYYMM.md` を作成し、その月の集計結果を記録する。
